# Quest (의뢰)

의뢰 정적 데이터를 담는 ScriptableObject. `Recipe`와 동일하게 `ItemEntryList`로 요구 품목을 표현한다.

## 핵심 원칙

### 티어 체인

> **티어 N 완성품 레시피는 반드시 티어 N−1 완성품을 재료로 포함한다.**

상위 티어 확장이 하위 라인을 **갈아엎지 않고** 그 위에 **얹히도록** 강제한다. 하위 완성품 라인은 상위 티어가 열린 뒤에도 계속 가동·소비된다.

### 생산과 의뢰 분리

- 의뢰는 생산을 **직접 트리거하지 않는다**.
- 공장은 항상 **재고 목표**를 향해 가동된다.
- 의뢰는 그 재고에서 **꺼내 가는** 방식으로만 연결된다.

```
재고 목표 설정 → 공장 가동 → 인벤 재고 축적 → 의뢰 납품(차감)
```

### 잉여 생산물

- 잉여 완성품은 **별도 판로**(매대 등)를 두지 않는다.
- **상시 의뢰**가 고정·변동 환율로 **무제한 납품**을 받아 **전량 흡수**한다.

### 골드 경제

- 후반 **골드 인플레이션**은 기존 **희귀 상인**(골드 싱크)이 흡수한다.
- 상인 판매 가격대도 **티어와 함께 기하급수**로 스케일해야 한다.

## 의뢰 종류

의뢰는 **일반** · **상시** · **스토리** 세 종류다. 공통적으로 생산 트리거가 아니라 **재고 소비처**로 동작한다.

| 종류 | 대상 | 수요 | 보상 | 역할 |
|------|------|------|------|------|
| **일반** | 개별 장비·완성품 | **유한** (재고 목표치까지) | 골드·명성 | 목표 재고를 채우면 **수요 종료** |
| **상시** | **모든** 완성품 | **무제한** | 단위당 소액 (티어↑ 기하급수) | 사이언스팩 + 잉여 **배출구** |
| **스토리** | 파트별 검증 품목 | 유한 (필수) | **없음** | 클리어 속도로 티어·난이도 **밸런스 센서** |

### 일반 의뢰

- **개별 장비** 등 특정 완성품에 대한 **유한 수요**.
- 플레이어가 설정한 **재고 목표치**까지 납품을 채우면 해당 의뢰(또는 수요 슬롯)는 **멈춘다**.
- 골드·명성의 주된 수입원. **납기**가 있으며, 기한 내 전량 납품 시 정상 보상.
- 미납 시 원래 명성 보상의 **50%**만 지급(또는 차감). 부분 납품 불가 ([01-core-loop.md](01-core-loop.md)).
- **명성** threshold에 따라 의뢰 풀 단계가 열린다.

### 상시 의뢰

- **모든 완성품**을 대상으로 **무제한** 반복 납품 가능.
- 티어가 오를수록 요구 **공정·보상**이 **기하급수**로 증가한다.
- 역할:
  1. **사이언스팩** — 상위 티어·테크 진행에 필요한 지속적 물량 소비
  2. **잉여 생산물 배출구** — 재고 목표를 넘긴 물량을 골드 등으로 환전
- 예: 철 판 10개 → 골드 5 (단위·환율은 티어·밸런스에 맞게 조정).
- 납기·수락 개념 없음. 결산(또는 전용 UI)에서 **배수 단위로 제출**.

### 스토리 의뢰

- **보상 없음.** **전용 레시피 없음** — 기존 생산 라인으로만 충족 가능해야 한다.
- 해당 파트에서 **티어·난이도가 적절한지** 판단하는 **밸런스 센서**.
  - 너무 **쉽게** 클리어 → 요구량·난이도 **상향**
  - 너무 **어렵거나** 반복 실패 → **하향**
- 스토리 전개·기계 해금 등 진행 보상은 의뢰 밖(이벤트·해금 테이블)에서 처리 ([04-story.md](04-story.md)).
- **반드시 수락**하며, 납기 내 미납·실패 시 **게임오버** (`isMandatory`).
- MVP 클리어 조건: 구현된 **모든 스토리 의뢰** 완료.

## 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `title` | `string` | 의뢰 제목 |
| `clientName` | `string` | 의뢰인 이름 |
| `content` | `string` | 의뢰 내용 (본문) |
| `requiredItems` | `ItemEntryList` | 납품해야 할 품목 목록 |
| `questType` | `enum` | *(예정)* `Story` · `Normal` · `Perpetual` — [의뢰 종류](#의뢰-종류) |
| `isMandatory` | `bool` | *(예정)* 스토리 의뢰 = `true` ([02-data-structure.md](02-data-structure.md)) |
| `stockTarget` | `int` | *(예정)* 일반 의뢰 — 목표 재고. 도달 시 수요 종료 |

## 에셋 생성

1. Project 창에서 우클릭
2. **Create → DungeonFront → Quest**
3. Inspector에서 필드 입력
4. `requiredItems.length` 설정 후 **Resize**, 각 `entries`에 `Item`과 `count` 지정

## JSON 로더 (예정)

의뢰 수가 늘어나면 `StreamingAssets/Quests/` 등에 JSON을 두고, 게임 시작 시 로드하는 방식으로 확장한다.
SO 클래스(`Quest`)는 그대로 두고, 로더가 JSON을 읽어 런타임 `Quest` 인스턴스를 만들거나 SO를 갱신한다.

### JSON 스키마

```json
{
  "questType": "normal",
  "title": "철광석 납품",
  "clientName": "대장장이",
  "content": "제련소에 철광석이 부족합니다. 10개만 구해 주세요.",
  "requiredItems": [
    { "itemId": "iron_ore", "count": 10 }
  ]
}
```

`questType`: `story` | `normal` | `perpetual` (상시). 상시 의뢰는 완성품별 **1회 납품 단위**·환율을 정의하며 무제한 반복 제출한다. 일반 의뢰는 **재고 목표치**(`stockTarget`, 예정)까지 유한 수요.

| JSON 필드 | SO 필드 | 비고 |
|-----------|---------|------|
| `questType` | `questType` | *(예정)* 기본값 `normal` |
| `title` | `title` | |
| `clientName` | `clientName` | |
| `content` | `content` | |
| `requiredItems[].itemId` | `entries[].item.id` | 문자열 ID로 `Item` 조회 필요 |
| `requiredItems[].count` | `entries[].count` | |

### 로더 구현 시 참고

- `Item`은 plain class이므로 JSON에서는 `itemId` 문자열로 참조하고, 로더에서 아이템 DB(레지스트리)로 `Item`을 해석한다.
- `ItemEntryList.length`와 `entries` 배열 길이를 맞춘 뒤 `Resize()` 없이 직접 채운다.
- 로더 스크립트는 `Assets/Scripts/Quest/QuestLoader.cs` 등으로 추가한다.
- JSON 파일 위치·로드 시점(초기화 씬, 메뉴 진입 등)은 구현 단계에서 정한다.
