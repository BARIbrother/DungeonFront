# 상시 의뢰

**역할**: Dev2 · **Week**: 4 · **Issue**: 06  
**선행**: 의뢰 시스템 구조 · 뒷산 의뢰 SO

## 1. 이 작업물

의뢰의 하위 종류인 **상시 의뢰**를 구현한다.  
데이터는 **Drive → Utility program** 폴더의 퀘스트 라인 `.html`에 있다.

**코드**: `Assets/Scripts/Quest/` (Quest SO · 납품 UI)

## 2. 규칙

| 항목 | 내용 |
|------|------|
| 수락 | 받지 않아도 됨. 항상 납품 가능 |
| 유지 | 납품해도 **계속 열림** |
| 기한 | **없음** |
| requirement | html에 적힌 요구 = **1회(단위) 보상 기준** |

## 3. 배수 납품 (슬라이더)

보유 재화 중 일부를 requirement **단위의 배수 n**으로 납품한다.  
요구 품목마다 같은 n을 쓴다. 보상도 html 보상의 **n배**.

예: requirement = 철 주괴 ×2 + 철 막대 ×1

| n | 납품량 | 보상 |
|---|--------|------|
| 1 | 철 주괴 2 + 철 막대 1 | html 보상 ×1 |
| 2 | 철 주괴 4 + 철 막대 2 | html 보상 ×2 |
| n | 철 주괴 2n + 철 막대 n | html 보상 ×n |

- UI: **슬라이더**로 n 선택 (보유량으로 가능한 최대 n까지)
- n은 요구 전 품목에 대해 **동일**해야 함
