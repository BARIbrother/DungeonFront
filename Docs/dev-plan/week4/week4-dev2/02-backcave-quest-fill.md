# 뒷산 의뢰 채우기

**역할**: Dev2 · **Week**: 4 · **Issue**: 02  
**선행**: W3 완성된 의뢰 시스템 구조 · Quest SO

## 1. 이 작업물

추가로 구현할 로직은 거의 없다.  
완성된 의뢰 시스템에 **의뢰 SO를 생성**해, 기획된 퀘스트를 채운다.

## 2. 기획 원본

의뢰 라인·필요 데이터는 **Drive → Utility program** 폴더의 퀘스트 라인 `.html`에 정리되어 있다.  
그 내용을 보고 SO 필드를 채운다.

## 3. 작업 내용

- 뒷산(`001xxxxx`) Quest SO 생성
- html에 있는 필수·일반 의뢰를 SO에 반영 (표시명·요구·납기·보상·필수 여부 등)
- 스토리 필수 / 일반 풀 구분
- 기존 수락·결산·납품 구조에 꽂아 Prepare에서 수락·결산 납품이 되게 함

## 4. 완료 기준

- [ ] Drive html 기준 뒷산 의뢰 SO 생성·필드 채움
- [ ] 필수·일반 구분 반영
- [ ] Prepare 수락·결산 납품 가능
